# 더바다 고객 퍼널-역할 재정의 텍스트 설명서 v1

## 1. 왜 이 문서를 보나
이 문서는 그림 없이도 고객 퍼널과 팀 역할을 이해하고 바로 실행할 수 있도록 정리한 실무용 문서다.

## 2. 핵심 한 줄
우리는 고객을 유입시키는 것보다, 유입된 고객을 신뢰 기반 소개로 재생산하는 구조를 먼저 안정화한다.

## 3. 단계별 흐름(S1~S9)
- S1 유입접수: 고객 정보/동의 확보
- S2 TM 1차분류: 가능/보류/불가 판단
- S3 DB 분류/배정: 담당자 배정 + 근거 로그
- S4 미팅 사전준비: 일정/자료/사전심사
- S5 미팅/진단: 후속 액션 결정
- S6 소개전환: 소개자 -> 신규 고객 S1 재진입
- S7 청구/보상 실행: 서류 완결 후 청구
- S8 보험설계/계약: 설계 제안 및 계약 전환
- S9 사후관리/재소개: 민원 대응 + 재소개 트리거

## 4. 역할 핵심
- 운영팀: 속도와 정확도
- 영업팀: 미팅 품질과 소개 전환
- 청구팀: 서류 완결성과 리드타임
- 준법감시: 게이트 준수
- 고객CS: 민원 해결과 루프 유지

## 5. 준법 게이트
- G1: 동의 없으면 시작 금지
- G2: 상태 변경 사유 기록 필수
- G3: 스크립트 핵심 문구 점검 필수
- G4: 필수서류 누락 시 청구 금지
- G5: 모집/보상 역할경계 위반 금지
- G6: 민원은 준법 자동 참조
