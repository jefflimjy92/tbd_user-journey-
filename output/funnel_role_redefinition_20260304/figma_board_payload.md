- B2 간편청구
  - Entry Trigger: 청구 편의 니즈
  - Owner Team: 운영팀/청구팀
  - KPI: 청구전환율
- B3 가족연동
  - Entry Trigger: 가족 보장 점검 요청
  - Owner Team: 영업팀
  - KPI: 가족연동율
- B4 소개유입
  - Entry Trigger: 기존고객 소개
  - Owner Team: 영업팀
  - KPI: 소개 DB 전환율

# Frame 2. 공통 행동단계
- S1 유입접수
  - Entry: 고객 접수 의사 및 기본정보 입력
  - Exit: 필수 동의 완료
  - Gate: G1
- S2 TM 1차분류
  - Entry: 동의 완료 DB
  - Exit: 미팅 가능/보류/불가 분류
- S3 DB 분류/배정
  - Entry: S2 분류 확정
  - Exit: 담당 배정 + 사유 로그
  - Gate: G2
- S4 미팅 사전준비
  - Entry: 배정된 미팅 대상
  - Exit: 일정/장소/준비자료 확정
- S5 미팅/진단
  - Entry: 미팅 확정
  - Exit: 후속액션 결정
  - Gate: G3
- S6 소개전환
  - Entry: 미팅 후 긍정 반응
  - Exit: 신규 소개 대상 S1 재진입
- S7 청구/보상 실행
  - Entry: 청구 필요 건 확정
  - Exit: 청구 접수 및 안내 완료
  - Gate: G4
- S8 보험설계/계약
  - Entry: 재설계 니즈 확인
  - Exit: 계약/미체결 사유 기록
  - Gate: G5
- S9 사후관리/재소개 루프
  - Entry: S7 또는 S8 종료
  - Exit: 사후관리 완료 + 재소개 트리거
  - Gate: G6

# Frame 3. RACI
- S1~S3: R 운영팀 TM 담당 / A 운영팀장 / C 데이터·IT / I 대표·영업팀장
- S4~S5: R 영업팀 상담담당 / A 영업팀장 / C 준법감시·청구팀 / I 운영팀
- S6: R 영업팀 상담담당 / A 영업팀장 / C 준법감시 / I 운영팀·경영진
- S7: R 청구팀 담당 / A 청구팀장 / C 영업 보상담당 / I 운영팀·고객CS
- S8: R 영업팀 설계담당 / A 영업팀장 / C 준법감시 / I 청구팀·경영진
- S9: R 고객CS/보상담당 / A 고객CS팀장 / C 영업·청구 / I 경영진

# Frame 4. 준법게이트
- G1(S1): 필수 약관/개인정보 동의 누락 시 진행 금지
- G2(S3): 상태값 변경 사유/근거 로그 필수
- G3(S5): 스크립트 준수 점검 항목 기록 필수
- G4(S7): 동의서/위임장 세트 완결성 필수
- G5(S8): 모집/보상 역할 경계 위반 탐지
- G6(S9): 민원/분쟁 건 준법감시 자동 참조

# Frame 5. 검증 시나리오
- T1: B1 고객이 S1~S9 추적 가능
- T2: B4 소개유입이 S6→S1 루프로 재진입
- T3: 계약 미발생 고객도 S7→S9 연결
- T4: S1/S5/S7/S8 게이트 증빙 누락 0
- T5: 동일 단계 R/A 충돌 없음
- T6: 모든 WorkItem에 오너/마감/완료기준 존재
